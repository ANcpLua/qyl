GET STARTED

01

Add qyl to docker-compose.yml

services: qyl: image: ghcr.io/ancplua/qyl:latest ports: ["5100:5100", "4317:4317"] volumes: [qyl_data:/data] your-app: environment: OTEL_EXPORTER_OTLP_ENDPOINT: http://qyl:4317
02

Restart

docker compose up -d

Open localhost:5100 — your telemetry is there.

01

Run qyl

docker run -d -p 5100:5100 -p 4317:4317 -v ~/.qyl:/data ghcr.io/ancplua/qyl:latest
02

Point your app

export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317

Open localhost:5100 — your telemetry is there.

One env var. Every language. Python, .NET, Node, Go, Java, Rust — all OpenTelemetry SDKs respect OTEL_EXPORTER_OTLP_ENDPOINT. No per-language setup.

CUSTOMIZE

Add env vars. Restart. That's the entire configuration model. All settings are optional — defaults work out of the box.

QYL_PORT = 5100
QYL_GRPC_PORT = 4317
QYL_TOKEN = (none)
QYL_DATA_PATH = ./qyl.duckdb
QYL_MAX_RETENTION_DAYS = 30
QYL_MAX_SPAN_COUNT = 1,000,000
# Change anything. Restart. Done. docker compose up -d
