## 1. Overview

`src/qyl.collector/` — ASP.NET Core application. Single process, single Docker image.

Responsibilities:

- Accept OTLP telemetry (gRPC + HTTP, protobuf + JSON)
- Decode, normalize, and store in DuckDB
- Stream live telemetry via SSE
- Serve REST API for dashboard, MCP, and external consumers
- Host dashboard static files in production

## 2. OTLP Ingestion

### 2.1 Endpoints

- gRPC on `QYL_GRPC_PORT` (default 4317)
- HTTP protobuf on `QYL_OTLP_PORT` (default 4318) at `/v1/traces`, `/v1/logs`, `/v1/metrics`
- HTTP JSON on same port (content-type negotiation)

### 2.2 Decode Pipeline

```text
OTLP bytes → protobuf decode → OtlpConverter → normalized spans/logs/metrics → DuckDB write
```

`OtlpConverter` normalizes gRPC and HTTP inputs into the same internal model. Resource attributes (`service.name`, etc.) are extracted and attached to every record.

### 2.3 Attribute Handling

Promoted columns: frequently-queried attributes get their own DuckDB columns (generated by `promoted-columns.g.sql`). All other attributes stored as MAP columns.

GenAI semantic conventions 1.40 attributes (`gen_ai.request.model`, `gen_ai.usage.input_tokens`, `gen_ai.usage.output_tokens`) are promoted columns.

### 2.4 Codex Telemetry

`CodexTelemetryMapper` handles OpenAI Codex-specific attribute mappings into standard GenAI semconv.

## 3. Storage

### 3.1 DuckDB

Single file database at `QYL_DATA_PATH`. Columnar storage optimized for analytical queries over time-series telemetry.

Tables:

- `spans` — trace spans with promoted attribute columns
- `logs` — structured log records
- `metrics` — metric data points
- `sessions` — user/agent sessions
- `issues` — error grouping and tracking
- `fix_runs` — autofix execution records
- `agent_runs` — agent invocation records
- `services` — service registry

### 3.2 Schema Control

`DuckDbSchema.SchemaControl.cs` manages migrations. SQL migration files in `Storage/Migrations/`. Schema version tracked in a control table.

### 3.3 Data Retention

`DataRetentionService` runs every 6 hours. `QYL_RETENTION_DAYS` environment variable (0 = disabled). Deletes old records, then CHECKPOINTs.

### 3.4 Reader Extensions

`DuckDbReaderExtensions` provides typed accessors for reading DuckDB results into C# types.

## 4. Realtime

### 4.1 SSE Streaming

Server-Sent Events stream live telemetry to the dashboard. Spans, metrics, and logs only — issues are REST-only.

`ObservationSubscription` + `SubscriptionManager` handle per-client subscriptions. `SchemaVersionNegotiator` handles client/server schema compatibility.

### 4.2 Live Deduplication

`LiveLogDeduplicator` prevents duplicate log entries in the SSE stream.

## 5. API

### 5.1 Endpoint Organization

Each feature area has its own `*Endpoints.cs` file with static `Map*` extension methods.

Major endpoint groups:

- Query endpoints (traces, spans, logs, metrics)
- Service endpoints (registry, classification)
- Issue endpoints (grouping, analytics)
- MCP-facing endpoints (sessions, logs, traces, metrics, API keys)
- Cost endpoints (pricing, aggregations, budget)
- Health endpoints

Exclusions: see `00-architecture.md` section 1.1 (scope) and section 5 (kill list).

### 5.2 Query Layer

`QueryEndpoints` — general telemetry queries.
`LogSummaryEndpoints` + `LogSummaryService` — log aggregation and summary.
`LogCursor` — cursor-based pagination for log streams.

### 5.3 Serialization

`QylSerializerContext` — source-generated JSON serializer context for AOT compatibility.

## 6. Auth

`TokenAuth` — API key validation for external access.

## 7. Services

`ServiceClassifier` — auto-classifies services based on telemetry attributes.
`ServiceEndpoints` — CRUD for service registry.

## 8. Definition of Done

- [x] gRPC and HTTP OTLP endpoints accept and decode all three signals
- [x] OtlpConverter produces identical internal model from both transports
- [x] Promoted columns match GenAI semconv 1.40 attribute names
- [x] SSE streams deliver spans/logs/metrics within 1 second of ingestion
- [x] DuckDB schema migrations apply idempotently
- [x] Data retention deletes records older than configured threshold
- [x] REST API returns paginated results (no unbounded queries)
- [x] Health endpoints report storage and ingestion status
- [x] Single Docker image serves dashboard + API + OTLP on configured ports
